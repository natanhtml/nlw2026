# Formato de Specs

## Estrutura

```
# Especificação - [Nome da Feature]

## Visão Geral
[Breve descrição do problema/solução]

## Requisitos Funcionais
- Requisito 1
- Requisito 2

## Análise de Alternativas
[Se aplicável - comparações de libs/abordagens]

## Arquitetura
### Estrutura de Arquivos
[Tree de arquivos]

### Fluxo de Dados
[Se aplicável]

## Implementação
### To-dos
- [ ] Tarefa 1
- [ ] Tarefa 2

### Comandos Úteis
```bash
# comandos relevantes
```
```

## Regras

- Começar sempre com "Especificação -"
- Incluir To-dos com checkboxes `[ ]`
- Adicionar decisões confirmadas no final quando houver
- Manter conciso - seção deve caber em uma tela
